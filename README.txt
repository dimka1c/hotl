1. Файлы для обработки должны находится в папке /data/
    Вы можете положить несколько файлов в эту папку, например
    Мониторы.xls, Ноутбуки.xls и т.д. В этом случае парсер будет
    последовательно обрабатывать файлы, т.е. сначала обработает мониторы, занесет изменения в файл,
    потом обработает ноутбуки и занесет изменения в файл и т.д.

2. Результат обработки так же находится в папке /data/
    Парсер все изменения заносит в те же файлы
    Т.е. если вы в /data/ положили файл Мониторы.xlsx, то после обработки
    результат будет в этой же папке в этом же файле.

3. Принцип работы
    - Входящие файлы должны быть загружены в папку /data/
    - Входящие файлы должны иметь раширение xls или xlsx
    - Файлов может быть несколько, парсер последовательно обрабатывает каждый файл
    - Из файла выбираются все товары, которые не имеют характеритик вообще или не все характеристики заполнены
    - Происходит поиск товаров на hotline.ua и спарсивание характеристик
    - Скачиваются фотографии товара и им задаются имена в соответствии с Артикулем товара, например: 199@1.jpg, 199@2.jpg, 199@3.jpg
    - После получения характеристик файл в папке /data/ обновляется.
    - Процесс парсинга товаров Вы можете наблюдать в отдельном окне терминала.

4. Результаты парсера:
    - папка /data/ - файлы с расширением xlsx
    - папка /image/ - фотографии товаров
    - папка /work/ - файл problem.txt - в этом файле хранятся все товары, которые по какой-либо причине не нашли на hotline.ua
    - При открытии результирующего файла, Вы можете увидеть, что ячейки имеют красный либо зеленый фон.
        - Зеленый - эта характеристика спарсилась
        - Красный - возникла проблема, при парсинге этой характеристики
        

5. Товары, котороые не найдены на hotline.ua или не определено, какой товар парсить,
    в файл результата не попадают. Парсер выдаст Вам сообщение, что товар не найден, или, если
    найдено более 1 товара и парсер не определил, какой качать - выдаст, что есть несколько вариантов
    для данного товара. (Нарпимер, нельзя определится, какой товар забирать с hotline.ua при
    поиске по такому наименованию, как "Geforece 5700")

6. Парсер максимально автоматизирован для обхода блокировок hotline.ua, однако допускается, что могут возникнуть
    ситуации, когда потребуется вручную добавить адреса прокси в файл proxy-reserve.json.
    Этот файл находится в папке /user_config/. Обычно достаточно порядка 5 адресов.
    Данный файл заполнен данными для Вашего удобства. Формат файла менять нельзя (т.н. нельзя доставлять пробелы,
    переносы строк, добавлять какие-либо символы в файл). Вам достаточно зайти на любой сайт с бесплатными прокси
    и прописать в файл ip-адрес, порт и протокол. Не рекомендуется работать с прокси по HTTP протоколу.
    Предпочтительный протокол SOCKS4, SOCKS5.
    Файл proxy-reserve.json желательно обновлять раз в сутки, потому как бесплатные прокси часто отказывают.
    В процессе работы парсер сам выберет прокси, с которыми он будет работать, этот проесс автоматизирован.

Желаем удачи в работе!
